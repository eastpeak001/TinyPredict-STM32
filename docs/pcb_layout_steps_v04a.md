# TinyPredict-STM32 V0.4A PCB Layout Steps

V0.4A 是基于 Blue Pill / STM32F103C8T6 开发板的扩展底板。布局目标是让当前 V0.3 原型从面包板迁移到更稳定的 PCB 载板，同时保留调试便利性。

## PCB 接口布局方案

建议布局方向如下：

| 区域 | 放置内容 | 设计理由 |
| --- | --- | --- |
| 中央 | Blue Pill / STM32F103C8T6 开发板插座 | 主控居中，方便向四周分配信号 |
| 左上 | 电源输入、稳压、电源开关、电源 LED | 电源入口明确，方便走 5V 和 3.3V 主干 |
| 右上 | SWD 接口 | 靠边放置，方便 ST-Link 下载和调试 |
| 左下 | USB-TTL 接口 | 靠边放置，方便连接串口线 |
| 前侧 | OLED 接口 | OLED 朝向用户，便于现场查看 RMS 和状态 |
| 板边 | MPU6050 接口 | 方便将传感器外接到风扇外壳或支架 |
| 右下 | 蜂鸣器接口和 LED | 报警和状态输出集中，方便观察和扩展 |

## 第 1 步：板框 + Blue Pill 插座

- 先确定 PCB 外形尺寸，预留安装孔。
- 将 Blue Pill / STM32F103C8T6 开发板插座放在 PCB 中央。
- 确认 USB 口、复位按键、BOOT 跳帽等不会被其他器件遮挡。
- 给开发板两侧排针保留足够焊接和插拔空间。

## 第 2 步：电源模块

- 在左上角放置 Type-C 6P(073) 或 5V 输入接口。
- 放置电源开关，保证 5V 输入先经过开关再进入稳压电路。
- 放置 AMS1117-3.3 或兼容 3.3V LDO。
- 在 LDO 输入和输出附近分别放置 10uF 电容。
- 在 3.3V 主干上放置电源指示 LED 和 1k 限流电阻。
- 电源走线适当加宽。

## 第 3 步：I2C OLED / MPU6050 接口

- OLED 接口放在 PCB 前侧，便于连接显示屏。
- MPU6050 接口放在板边，便于传感器线束引出到风扇。
- PB6 / I2C1_SCL 同时连接 MPU6050 SCL 和 OLED SCL。
- PB7 / I2C1_SDA 同时连接 MPU6050 SDA 和 OLED SDA。
- 预留 4.7k I2C 上拉电阻焊盘，可根据模块是否自带上拉决定是否焊接。
- 在 MPU6050 接口附近放置 0.1uF 去耦电容。

## 第 4 步：USB-TTL 接口

- USB-TTL 接口放在左下或板边位置。
- PA9 / USART1_TX 连接 USB-TTL RX。
- PA10 / USART1_RX 连接 USB-TTL TX。
- 接口丝印清楚标注 3V3、GND、TX、RX。
- 该接口只用于串口调试和 Python 上位机采集，不用于给风扇供电。

## 第 5 步：SWD 接口

- SWD 接口放在右上或其他靠边位置。
- PA13 连接 SWDIO。
- PA14 连接 SWCLK。
- 同时引出 3V3 和 GND。
- 接口方向和丝印要清楚，避免 ST-Link 反接。

## 第 6 步：LED 和蜂鸣器预留

- 状态 LED 与 PC13 / STATUS_LED 保持一致。
- 电源 LED 接 3.3V，通过 1k 限流电阻到 LED，再到 GND。
- 蜂鸣器 HYT-0903 可放在右下角，或先只预留接口。
- 蜂鸣器控制信号可暂命名为 BUZZER_CTRL，后续固件版本再定义具体 GPIO。

## 第 7 步：PCB 布局

- Blue Pill 居中，外设接口围绕四周展开。
- 电源模块靠近输入接口，减少 5V 走线长度。
- OLED 接口朝向用户一侧。
- MPU6050 接口靠板边，减少传感器线束绕行。
- USB-TTL 和 SWD 都放在边缘，方便调试。
- 传感器线和电机电源线分开，避免电机噪声耦合。
- 保留安装孔，方便后续外壳和风扇测试平台固定。

## 第 8 步：布线和 DRC 检查

- 先布电源线，再布 I2C、USART1、SWD 和 LED 信号线。
- 5V 和 3.3V 走线适当加宽。
- GND 尽量使用铺铜，并确保各接口地连接可靠。
- I2C 线尽量短，避免与电机电源线平行长距离走线。
- 检查 Type-C 供电、LDO 输入输出、电容极性和 LED 极性。
- 检查 USB-TTL TX/RX 是否交叉正确。
- 检查 SWDIO / SWCLK 是否连接到 PA13 / PA14。
- 运行 DRC，修复间距、未连接网络、短路和丝印压焊盘等问题。

## 第一版风险控制

- V0.4A 不直接焊 STM32F103C8T6 芯片，降低最小系统失败风险。
- V0.4A 不直接焊 MPU6050 芯片，优先使用已验证模块。
- 风扇电机不要从本 PCB 取电，风扇应单独供电。
- PCB 只负责监测、显示和通信。
- 如果测试中出现 RMS 异常跳变，应优先检查传感器固定、I2C 线长、电机噪声和供电稳定性。
